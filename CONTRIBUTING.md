# Contributing to Nirantar Health

First off, thank you for considering contributing to Nirantar Health. It's people like you that make open-source health tech such a great tool for social impact.

## Where do I go from here?
If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request before taking the time to write code.

## Fork & create a branch
If this is something you think you can fix, then fork Nirantar Health and create a branch with a descriptive name.

## Implement your fix or feature
At this point, you're ready to make your changes! Feel free to ask for help; everyone is a beginner at first. 

## Code Style
- Follow the Vanilla JS patterns already established in `app.js`.
- Keep CSS changes within `styles.css` using the existing CSS variables.
- Ensure no external CDNs block the offline capabilities of the PWA.

## Make a Pull Request
When you're finished, push your branch up to your fork and create a Pull Request against the `master` branch of the main repository.
